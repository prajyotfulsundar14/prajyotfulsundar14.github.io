name: DevSecOps Pipeline
on: [push, pull_request]

jobs:
  playwright-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install deps
        run: npm ci
      - name: Run Playwright Tests
        run: npx playwright test --reporter=html

  semgrep-sast:
    runs-on: ubuntu-latest
    container: semgrep/semgrep
    steps:
      - uses: actions/checkout@v4
      - run: semgrep ci --config=p/owasp-top-ten

  snyk-sca:
    runs-on: ubuntu-latest
    steps:
      - uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          args: --severity-threshold=high

  trivy-container:
    runs-on: ubuntu-latest
    steps:
      - uses: aquasecurity/trivy-action@master
        with:
          image-ref: myapp:${{ github.sha }}
          severity: CRITICAL,HIGH
          exit-code: 1

  owasp-zap-dast:
    runs-on: ubuntu-latest
    steps:
      - uses: zaproxy/action-full-scan@v0.9.0
        with:
          target: https://staging.myapp.com

  deploy:
    needs: [playwright-tests, semgrep-sast, snyk-sca, trivy-container, owasp-zap-dast]
    runs-on: ubuntu-latest
    steps:
      - name: Deploy via ArgoCD
        run: argocd app sync my-app --grpc-web

INC-001 · CI/CD Pipeline Block: Critical CVE in lodash
SCASnykCI/CD
CRITICAL 2024-11-14 +
💥 Impact: Blocked 3 PRs across 2 microservices. Release delayed 4 hours.
🔎 Root Cause: Snyk detected CVE-2024-21538 in lodash@4.17.20 (CVSS 9.1). Pipeline halted at SCA stage automatically.
✅ Fix Applied: Upgraded lodash to 4.17.21. Added .snyk policy. Pipeline green in 12 minutes.
INC-002 · Auth Token Exposed in Playwright CI Logs
PlaywrightSecretsSAST
HIGH 2024-07-08 +
💥 Impact: 12 Playwright tests failed. Bearer token exposed in CI build logs shipped to Datadog.
🔎 Root Cause: Auth token hardcoded in Playwright test fixture. Log masking not configured on Datadog shipper.
✅ Fix Applied: Tokens migrated to GitHub Secrets. Log masking regex added. Playwright storageState used for auth.
INC-003 · Kubernetes Pod Compromise via Wildcard RBAC
KubernetesRBACOPA
CRITICAL 2024-05-19 +
💥 Impact: Privilege escalation to cluster-admin. 2 production namespaces exposed. P0 incident declared.
🔎 Root Cause: ServiceAccount had wildcard verb on all K8s resources. OPA Gatekeeper not enforced pre-deploy.
✅ Fix Applied: Least-privilege RBAC applied. OPA Gatekeeper constraints enforced. Trivy added as mandatory pre-deploy gate. Resolved in 14 minutes.
INC-004 · SQL Injection Found via OWASP ZAP DAST
DASTOWASP ZAPSQL Injection
MEDIUM 2024-03-05 +
💥 Impact: P95 API latency spiked to 4.2s. OWASP ZAP active scan found SQL injection in /api/search endpoint.
🔎 Root Cause: Raw SQL string concatenation in legacy search handler. DAST not present in pipeline at the time.
✅ Fix Applied: Parameterized queries implemented. OWASP ZAP DAST added as mandatory pipeline stage. Playwright perf assertions added.
